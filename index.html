MY API Documentation

Overview

This API manages user infoirmation, income records, and expense records with full CRUD operations. It serves JSON data through RESTful endpoints.

Notes

  • The API responds with JSON data only.
  • Authentication is currently not implemented (update this if you add auth).
  • Make sure your backend server is running (http://localhost:3000/) before making requests.
    •

API Endpoints

1. Users

  • GET /users — Retrieve all users
  • POST /users — Create a new user.
  • PUT /users/:id — Update a user by ID.
  • DELETE /users/:id — Delete a user by ID.

2. Income

  • GET /income — Retrieve all income records
  • POST /income — Create a new income record.
  • PUT /income/:id — Update income by ID.
  • DELETE /income/:id — Delete income by ID.

3. Expenses

  • GET /expenses — Retrieve all expense records
  • POST /expenses — Create a new expense record.
  • PUT /expenses/:id — Update expense by ID.
  • DELETE /expenses/:id — Delete expense by ID.
